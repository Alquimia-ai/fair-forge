# Agentic Metric

The Agentic metric evaluates AI agent performance at the **conversation level**. A conversation is correct only if **ALL** its interactions are correct. pass@K and pass^K are computed globally across all evaluated conversations using the agent's overall success rate `p = c/n`.

## Overview

- **Conversation Correctness**: All-or-nothing — one wrong turn fails the whole conversation
- **pass@K**: `1 - (1-p)^k` — probability of ≥1 correct conversation in k attempts
- **pass^K**: `p^k` — probability of all k conversations being correct
- **Tool Correctness**: Four-dimensional score per interaction — selection, parameters, sequence, utilization
- **Statistical Modes**: Frequentist (point estimate) or Bayesian (credible intervals)

### Formulas

```
p       = c / n                 # Global success rate (fully correct conversations / total)
pass@k  = 1 - (1 - p)^k        # P(≥1 correct in k independent attempts)
pass^k  = p^k                   # P(all k attempts correct)
```

<Note>
`p` is computed **globally** across all evaluated conversations — not per conversation. `k` is a required parameter representing how many independent attempts to model.
</Note>

## Basic Usage

<CodeGroup>
```python Frequentist (default)
from fair_forge.metrics.agentic import Agentic
from langchain_groq import ChatGroq
from your_retriever import AgenticRetriever

judge_model = ChatGroq(model="llama-3.3-70b-versatile", api_key="your-api-key", temperature=0.0)

result = Agentic.run(
    AgenticRetriever,
    model=judge_model,
    k=3,
    threshold=0.7,
)

# Single global result
print(f"Conversations: {result.c}/{result.n} fully correct  (p = {result.p:.1%})")
print(f"pass@{result.k} = {result.pass_at_k:.4f}")
print(f"pass^{result.k} = {result.pass_pow_k:.4f}")
```

```python Bayesian
from fair_forge.metrics.agentic import Agentic
from fair_forge.statistical import BayesianMode
from langchain_groq import ChatGroq
from your_retriever import AgenticRetriever

judge_model = ChatGroq(model="llama-3.3-70b-versatile", api_key="your-api-key", temperature=0.0)

result = Agentic.run(
    AgenticRetriever,
    model=judge_model,
    k=3,
    threshold=0.7,
    statistical_mode=BayesianMode(mc_samples=5000, ci_level=0.95),
)

print(f"pass@{result.k} = {result.pass_at_k:.4f}  [{result.pass_at_k_ci_low:.4f}, {result.pass_at_k_ci_high:.4f}]")
print(f"pass^{result.k} = {result.pass_pow_k:.4f}  [{result.pass_pow_k_ci_low:.4f}, {result.pass_pow_k_ci_high:.4f}]")
```
</CodeGroup>

## Built-in Display Methods

`Agentic` ships four static methods for zero-boilerplate result inspection:

```python
# Print global stats + per-conversation breakdown
Agentic.summary(result)

# Plot pass@K and pass^K curves (optionally save for papers/reports)
Agentic.plot(result, k_max=10, save_path="figures/passk_curves.png")

# Print pass@K and pass^K table for K = 1..k_max
Agentic.compare_k(result, k_max=10)

# Plot per-conversation tool correctness breakdown
Agentic.plot_tools(result, save_path="figures/tool_correctness.png")
```

## Parameters

### Required Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `retriever` | `Type[Retriever]` | Data source class — each Dataset = 1 conversation |
| `model` | `BaseChatModel` | LangChain-compatible model for LLM-as-judge evaluation |
| `k` | `int` | Number of independent attempts for pass@K/pass^K computation |

### Optional Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `statistical_mode` | `StatisticalMode` | `FrequentistMode()` | Statistical computation mode |
| `threshold` | `float` | `0.7` | Answer correctness threshold (0.0–1.0) |
| `tool_threshold` | `float` | `1.0` | Tool correctness threshold (0.0–1.0) |
| `tool_weights` | `dict[str, float]` | `0.25` each | Weights for tool dimensions (selection, parameters, sequence, utilization) |
| `use_structured_output` | `bool` | `False` | Use LangChain structured output (not supported by all providers) |
| `verbose` | `bool` | `False` | Enable verbose logging |

## Statistical Modes

<Tabs>
  <Tab title="Frequentist">
    Computes `p = c/n` as a point estimate and plugs it directly into the pass@K formulas.

    ```python
    # 2 fully correct out of 3 conversations, k=2:
    # p = 2/3 = 0.667
    # pass@2 = 1 - (1 - 0.667)^2 = 0.889
    # pass^2 = 0.667^2 = 0.444
    ```

    `pass_at_k_ci_low`, `pass_at_k_ci_high`, `pass_pow_k_ci_low`, `pass_pow_k_ci_high` are all `None`.
  </Tab>
  <Tab title="Bayesian">
    Places a Beta prior on `p`, samples from the posterior `p | c,n ~ Beta(c+1, n-c+1)`, and propagates uncertainty through the pass@K formulas to produce credible intervals.

    ```python
    # 2 correct out of 3 conversations, k=2, Beta(1,1) prior:
    # Posterior: Beta(3, 2) — centered near 0.667 but with uncertainty
    # pass@2 samples → mean=0.889, CI=[0.35, 0.99]
    # pass^2 samples → mean=0.444, CI=[0.08, 0.85]
    ```

    Recommended when `n < 20` conversations — the wide CI honestly communicates that more data is needed.

    ```python
    from fair_forge.statistical import BayesianMode

    result = Agentic.run(
        AgenticRetriever,
        model=judge_model,
        k=3,
        statistical_mode=BayesianMode(
            mc_samples=5000,
            ci_level=0.95,
            beta_prior_a=1.0,
            beta_prior_b=1.0,
        ),
    )
    ```
  </Tab>
</Tabs>

## Data Requirements

Each `Dataset` represents one complete conversation. Provide `agentic` and `ground_truth_agentic` on `Batch` objects to enable tool correctness scoring:

```python
from fair_forge.core.retriever import Retriever
from fair_forge.schemas.common import Dataset, Batch

class AgenticRetriever(Retriever):
    def load_dataset(self) -> list[Dataset]:
        return [
            Dataset(
                session_id="conversation_001",
                assistant_id="agent_v1",
                language="english",
                context="Math calculator conversation",
                conversation=[
                    Batch(
                        qa_id="q1_t1",
                        query="What is 5 + 3?",
                        assistant="The result is 8.",
                        ground_truth_assistant="8",
                        agentic={
                            "tools_used": [{
                                "tool_name": "calculator",
                                "parameters": {"a": 5, "b": 3},
                                "result": 8,
                                "step": 1
                            }],
                            "final_answer_uses_tools": True
                        },
                        ground_truth_agentic={
                            "expected_tools": [{
                                "tool_name": "calculator",
                                "parameters": {"a": 5, "b": 3},
                                "step": 1
                            }],
                            "tool_sequence_matters": True
                        }
                    ),
                    Batch(
                        qa_id="q1_t2",
                        query="What is 100 / 4?",
                        assistant="100 divided by 4 is 25.",
                        ground_truth_assistant="25"
                    ),
                ],
            ),
        ]
```

### Tool Data Structure

**`agentic`** (actual tool usage in the response):
```python
{
    "tools_used": [{"tool_name": "calculator", "parameters": {"a": 5, "b": 3}, "result": 8, "step": 1}],
    "final_answer_uses_tools": True
}
```

**`ground_truth_agentic`** (expected tool usage):
```python
{
    "expected_tools": [{"tool_name": "calculator", "parameters": {"a": 5, "b": 3}, "step": 1}],
    "tool_sequence_matters": True
}
```

## Output Schema

### AgenticMetric

Global result returned by `Agentic.run()`:

```python
class AgenticMetric(BaseModel):
    k: int                          # Attempts used for pass@K computation
    n: int                          # Total conversations evaluated
    c: int                          # Fully correct conversations
    p: float                        # Global success rate = c/n
    pass_at_k: float                # 1 - (1-p)^k
    pass_pow_k: float               # p^k
    pass_at_k_ci_low: float | None  # Lower CI — Bayesian only
    pass_at_k_ci_high: float | None # Upper CI — Bayesian only
    pass_pow_k_ci_low: float | None # Lower CI — Bayesian only
    pass_pow_k_ci_high: float | None# Upper CI — Bayesian only
    conversations: list[AgenticConversation]
```

### AgenticConversation

Per-conversation detail, accessible via `result.conversations`:

```python
class AgenticConversation(BaseModel):
    session_id: str
    assistant_id: str
    total_interactions: int
    correct_interactions: int
    is_fully_correct: bool
    threshold: float
    correctness_scores: list[float]
    correct_indices: list[int]
    tool_correctness_scores: list[ToolCorrectnessScore | None]
```

### ToolCorrectnessScore

```python
class ToolCorrectnessScore(BaseModel):
    tool_selection_correct: float  # Correct tools chosen
    parameter_accuracy: float      # Correct parameters passed
    sequence_correct: float        # Correct order (if required)
    result_utilization: float      # Tool results used in final answer
    overall_correctness: float     # Weighted average of the four dimensions
    is_correct: bool               # overall_correctness >= tool_threshold
    reasoning: str | None
```

## Interpretation

### Deployment Readiness

| pass@1 | pass^3 | Classification |
|--------|--------|----------------|
| > 0.90 | > 0.70 | Production-ready |
| > 0.90 | 0.30–0.70 | Functional but inconsistent |
| 0.70–0.90 | — | Needs improvement |
| < 0.70 | — | Not ready |

### pass@K vs pass^K

| Metric | Answers |
|--------|---------|
| `pass_at_k` | "If I run the agent K times, what's the chance of getting at least one correct result?" |
| `pass_pow_k` | "What's the chance the agent is correct every single time over K requests?" |

**Example — `p = 0.667`:**

| K | pass@K | pass^K |
|---|--------|--------|
| 1 | 0.667 | 0.667 |
| 2 | 0.889 | 0.444 |
| 3 | 0.963 | 0.296 |
| 5 | 0.996 | 0.132 |

The gap between pass@K and pass^K is the reliability inconsistency — a wide gap means the agent can succeed but not consistently.

### Tool Correctness

| Dimension | What it measures |
|-----------|-----------------|
| Selection | Right tools were called |
| Parameters | Correct arguments were passed |
| Sequence | Tools were called in the right order |
| Utilization | Tool results appear in the final answer |

## Computing Additional K Values

```python
from fair_forge.metrics.agentic import pass_at_k, pass_pow_k

for k in [1, 3, 5, 10]:
    pak = pass_at_k(result.n, result.c, k)
    ppk = pass_pow_k(result.n, result.c, k)
    print(f"K={k}: pass@K={pak:.3f}  pass^K={ppk:.3f}")
```

## Best Practices

<AccordionGroup>
  <Accordion title="Use Bayesian Mode for Small Test Suites">
    With fewer than 20 conversations, frequentist pass@K estimates can be misleading. A `p = 0.80` from 5 conversations has a 95% CI of roughly [0.38, 0.97] — Bayesian mode surfaces this uncertainty so you don't over-conclude from limited data.
  </Accordion>

  <Accordion title="Choose K to Match Your Deployment">
    - **K=1**: What fraction of single attempts succeed?
    - **K=3–5**: Best for most production scenarios (retry budget)
    - **K=10+**: High-stakes deployments requiring near-certain success
  </Accordion>

  <Accordion title="Set Meaningful Thresholds">
    - **Strict (0.8–0.9)**: Factual accuracy critical (medical, legal, finance)
    - **Moderate (0.7)**: General purpose — recommended default
    - **Lenient (0.6)**: Creative or subjective tasks
  </Accordion>

  <Accordion title="Define Complete Tool Ground Truth">
    Always provide `expected_tools` with exact parameter values and `tool_sequence_matters` set explicitly. Ambiguous ground truth produces unreliable tool correctness scores.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Judge Returns 0.0 for Correct Answers">
    The model may not be following the output format. The metric retries up to 3 times automatically. If scores remain 0.0, enable `verbose=True` to inspect judge responses and consider switching to a larger model.
  </Accordion>

  <Accordion title="Tool Correctness Always Fails">
    The default `tool_threshold=1.0` requires perfect tool execution. Lower it with `tool_threshold=0.75` to allow minor deviations. Verify tool names match exactly (case-sensitive) between `tools_used` and `expected_tools`.
  </Accordion>

  <Accordion title="use_structured_output Raises 400 Error">
    Some providers (e.g. Groq with certain models) do not support `response_format: json_schema`. Keep `use_structured_output=False` (the default) — the metric uses XML-delimited JSON extraction instead.
  </Accordion>

  <Accordion title="Bayesian CI is Very Wide">
    A wide CI means not enough data to estimate the true success rate precisely. This is correct behavior — collect more test conversations to narrow it. A CI of [0.2, 0.9] on 5 conversations honestly reflects the uncertainty.
  </Accordion>
</AccordionGroup>
